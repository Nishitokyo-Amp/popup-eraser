# Popup Eraser

サイトに出てくる**ポップアップ・ライトボックス・モーダル**を「クリック1つ」で**HTMLごと**削除する Chrome 拡張機能です。
ただ ✕ ボタンを押すだけだと再表示される場合があるため、DOM レベルで丸ごと削除します。

## 主な特徴

- **手動クリック削除モード**: 拡張機能アイコンから起動 → 消したい要素をクリックするだけ
- **賢いコンテナ検出**: クリック位置から `position: fixed` / `z-index` を遡って「ポップアップ本体」を自動特定
- **背景オーバーレイも同時削除**: 暗い半透明のオーバーレイ部分も検出して一緒に消去
- **スクロールロック自動解除**: `body { overflow: hidden }` 等で固定されたスクロールを復元
- **サイト別 ON/OFF**: サイトごとに有効/無効を記憶
- **Cookie同意バナー保護**: 法的なバナーを誤削除しないように選択可能
- **キーボードショートカット**: `Alt + Shift + E` でモード起動

## インストール手順（開発者モード）

1. Chrome のアドレスバーに `chrome://extensions` を開く
2. 右上の「**デベロッパー モード**」を ON
3. 「**パッケージ化されていない拡張機能を読み込む**」をクリック
4. このフォルダ（`Popup-Blocker`）を選択
5. ツールバーにハサミ ✂︎ アイコンが追加されれば完了

## 使い方

1. ポップアップが邪魔しているページで拡張機能アイコンをクリック
2. **「🎯 クリック削除モードを開始」** を押す（または `Alt + Shift + E`）
3. 画面上部に「クリック削除モード」のバーが表示される
4. 消したいポップアップにマウスを乗せると **赤枠** でハイライト
5. クリックすると HTML ごと削除され、スクロールロックも解除される
6. 終了したいときは **`ESC`** または **右クリック**

### 設定項目

| 設定 | 説明 |
|------|------|
| このサイトで有効 | 現在のサイトでのみ拡張機能をON/OFF |
| Cookie同意バナーを保護 | 「同意」「Cookie」等を含む要素を削除対象から除外（緑の枠で表示） |
| 自動削除モード | ON: ポップアップ表示と同時に外側ラッパーを自動検出して削除。OFF: ⌥⇧Eでの手動クリック削除のみ。 |

### 自動削除モードの挙動

ONにすると、ページ読込時およびDOM変更時に下記の条件を満たす要素を「外側のラッパー」まで遡って自動で非表示化します:

- `<dialog open>` / `role="dialog"` / `aria-modal="true"`
- 画面の30%以上を覆う、クラス名にmodal/dialog/popup/overlay/lightboxを含む要素
- `body` 直下の `position: fixed` で画面の40%×30%以上を覆う高z-index要素

誤って消した場合は **`⌘+Z`** で復元できます。復元した要素は60秒間は自動削除の対象外になります。

## 「削除」の実装方式について

物理的に `node.remove()` で DOM から削除すると、React / Vue / Next.js などのフレームワークが内部で管理している仮想DOMと不整合を起こし、後で `NotFoundError: Failed to execute 'removeChild'` などのエラーを起こすサイトがあります（TVer など）。

Popup Eraser は代わりに、以下を **`!important`** で要素に適用してフレームワーク安全に「消した」ように見せます：

```
display: none !important;
visibility: hidden !important;
pointer-events: none !important;
```

これは「✕ボタンを押す」のとは違い、サイト側の click イベントを起動しないため「閉じてもまた表示される」現象も起きません。ユーザーから見ても完全に消えますし、`Ctrl+Z` / `⌘+Z` でスタイルを戻すだけで瞬時に復元できます。

## 削除アルゴリズムの概要

`content.js` 内 `findPopupContainer()` は、クリックされた要素から親方向へ最大25段遡り、以下の条件に当てはまるノードを「ポップアップコンテナ」として記憶します。

- `position` が `fixed` / `sticky` / `absolute` で、かつ `z-index` が10以上
- もしくは画面の50%×30%以上を占有している `fixed`/`absolute` 要素

最も内側で「最後にマッチした祖先」を削除対象とすることで、ボタンや画像をクリックしても**ポップアップ本体**が消えるようになっています。

加えて `findRelatedOverlays()` が、画面全体を覆う `position: fixed` の半透明要素（典型的なライトボックスの黒背景）を別途検出して同時削除します。

最後に `releaseScrollLock()` が、`html` / `body` の `overflow:hidden`、`position:fixed`、`touch-action` を上書き解除し、スクロールが効くページに戻します。

## 制限事項

- `chrome://`, Chrome ウェブストア, `file://` の一部ページなど、Chrome が拡張機能の注入を禁止しているページでは動作しません
- 一部のSPAでは、削除後にサイト側が同じノードを再生成することがあります（その場合は再度クリック）
- **動画プレイヤー内部のモーダル** (TVer/video.js の `vjs-modal-dialog` など) を削除すると、プレイヤー側がエラー扱いになって「このコンテンツは現在表示できません」などの後発エラーが出ることがあります。これはサイト側のロジック由来のため、誤って削除した場合は **`Ctrl+Z` / `⌘+Z`** で直前の削除を元に戻せます。
- ポップアップが「視聴の前にお願い」「年齢確認」などの **ゲート**として機能している場合、削除しても再生は始まりません（サイト側の状態管理がユーザー操作を待っているため）。

## キーボードショートカット

| キー | 動作 |
|------|------|
| `Alt + Shift + E` | クリック削除モードのON/OFF |
| `ESC` | モード終了 |
| `右クリック` | モードキャンセル |
| `Ctrl+Z` / `⌘+Z` | 直前の削除を元に戻す（最大20回） |

## 開発

特別なビルドステップは不要です。ファイルを編集したら `chrome://extensions` の更新ボタン ↻ を押すだけで反映されます。

## ライセンス

個人利用向け。
